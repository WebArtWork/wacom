Project Overview

Package

wacom

Common Angular building blocks for shared app behavior, with browser-only APIs guarded so SSR stays intact.

License

MIT

Published as a public package and designed for reuse across multiple apps.

Installation

Install the package

$ npm i --save wacom

Usage

Bootstrap with provideWacom()

Use standalone providers by default. WacomModule remains available for older NgModule apps, but the intended entry point is provideWacom. 

import { provideWacom } from 'wacom';

export const appConfig = {
  providers: [provideWacom()],
};

Configuration

Centralize app-level defaults

Configure library-wide behavior once instead of scattering base URLs, storage prefixes, or route metadata policy across components. 

import { provideWacom } from 'wacom';

export const appConfig = {
  providers: [
    provideWacom({
      http: { url: 'https://api.example.com' },
      store: { prefix: 'waStore' },
      meta: {
        useTitleSuffix: false,
        applyFromRoutes: true,
        defaults: { links: {} },
      },
      network: {},
      socket: false,
      io: undefined,
    }),
  ],
};

Services

Core building blocks

CoreService

Low-level SSR-safe helpers for ids, copy, viewport, device, locks, and signals.

HttpService

Shared base URL, header management, callbacks, and Observable requests on top of HttpClient.

StoreService

Async-first persistence wrapper for localStorage or a custom storage provider.

MetaService

Centralized page meta handling for route-driven and manual SEO updates.

CrudService

Reusable document data layer with local cache, signals, offline queueing, and lifecycle hooks.

EmitterService

Hot event channels and task-completion coordination built on signals and RxJS.

NetworkService

Signal-driven connectivity classification with active endpoint probing.

ThemeService

Appearance preference manager for mode, density, radius, and theme cycling.

TranslateService

Runtime translation registry backed by signals with config-based language bootstrapping.
